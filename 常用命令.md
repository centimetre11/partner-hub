## 一、最常用（日常三板斧）

### 1. Mac 拉最新代码（手机/别人合并到 main 之后）

```bash
cd "/Users/chenmin/Desktop/Supplier management/partner-hub"
git pull origin main
npx prisma generate    # schema 有变时跑
npx prisma db push     # 本地数据库结构要更新时跑
```

### 2. Mac 改完代码推上去

```bash
cd "/Users/chenmin/Desktop/Supplier management/partner-hub"
git status
git add .
git commit -m "简要说明改了什么"
git push origin main
```

### 3. 部署到线上（合并到 main 之后）

```bash
cd "/Users/chenmin/Desktop/Supplier management/partner-hub"
./scripts/deploy.sh ubuntu@43.164.65.54
```

> 这条命令会在服务器上 `git pull` + 重建 Docker，约 2～3 分钟。  
> `.env` 不会被覆盖，数据库和上传文件会保留。

## 三、Git / GitHub

查看状态：

```bash
git status
git log --oneline -10
```

首次配置 GitHub（只需跑一次）：

```bash
./scripts/setup-github.sh
```

查看 GitHub 上的 PR：

```bash
gh pr list --repo centimetre11/partner-hub
```

## 四、服务器运维

SSH 登录服务器：

```bash
ssh ubuntu@43.164.65.54
```

登录后常用：

```bash
cd /opt/partner-hub
git log --oneline -5
docker compose ps
docker compose logs -f --tail=50 partner-hub
```

手动在服务器上更新（等价于 deploy.sh，一般不用手跑）：

```bash
cd /opt/partner-hub
git pull --ff-only
docker compose up -d --build
```

重启容器（不拉代码）：

```bash
cd /opt/partner-hub
docker compose restart partner-hub
```

## 五、首次部署（已有环境可跳过）

服务器第一次装（需要本地 `.env` 已配好）：

```bash
./scripts/deploy.sh ubuntu@43.164.65.54 --domain camelusai.com --init
```

## 六、推荐工作流

```
改代码（Mac 或 手机 Cursor）
    ↓
合并到 main（GitHub PR 或 git push）
    ↓
Mac: git pull origin main
    ↓
Mac: ./scripts/deploy.sh ubuntu@43.164.65.54
    ↓
打开 https://camelusai.com/ 验证
```

手机 Cursor 改完 → 在手机上说「合并 PR 到 main」→ Mac 执行上面「拉代码 + 部署」两步即可。
