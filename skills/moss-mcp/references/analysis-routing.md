# 分析路由

先识别用户真正目标，再选择方法论。这里抽取 Moss 的稳定分析工作法，不照搬内部提示词，也不固化当前工具清单。

路由只决定“读哪份方法论”和“用什么分析框架”。真正可调用能力以实时 `tools/list` 为准。

## 客户洞察

进入条件：用户输入企业 / 机构名称，或询问企业信息、规模、特定维度、对标、名单筛选、产业链潜客、CRM 合作、销售切入。

读取：`methodologies/customer-insight.md`

路由核心：先定 `primary_scene`，再按场景执行。不要把所有企业问题都套成企业档案，也不要为了全面自动铺开所有企业维度。

- `company_overview`：单企业画像、档案、销售跟进材料。
- `company_scale`：营收、人数、注册资本、体量变化、规模对比。
- `company_multidim`：专利、股东、诉讼、联系方式、新闻等点名维度。
- `company_similar`：有参照企业的相似 / 对标。
- `company_filter`：按行业、地区、规模、标签筛企业名单。
- `chain_prospect`：从产品 / 产业链出发探索还能卖给谁。
- `batch_file_query`：文件或多企业名单批量查询。
- `crm_query`：我方合作、合同、责任销售、CRM 客户筛选。

## 风险洞察

进入条件：用户询问供应商准入、客户信用、投资尽调、合规、司法、行政处罚、经营稳定性、风险等级。

读取：`methodologies/risk-insight.md`

路由核心：按 Phase 1-5 执行，先锁定评估对象和边界，再采集、校准、定档。风险不是字段罗列，必须解释事实如何影响业务场景。

- 浅度：快速风险画像，但仍要有事实触发原因。
- 深度：准入、授信、尽调、投资等支撑决策，做九维扫描和必要增强。
- 监控：持续跟踪、预警、订阅，输出监控对象、触发器和复核动作。
- 精简判断：只需要风险字段时，输出 `risk_level`、触发原因、影响路径、置信度和建议动作，不写完整报告。

## 舆情洞察

进入条件：用户询问舆情、口碑、负面、竞品动态、声量趋势、市场声音、危机信号。

读取：`methodologies/opinion-insight.md`

路由核心：按任务分成检索、危机识别、趋势洞察和成稿四类步骤，不把它们混成一个动作。这里是外部分析步骤，不代表外部存在同名角色或工具。

- 检索：设计关键词、时间窗、来源和精读对象。
- 危机识别：识别负面、危机、攻击论点、传播源和 L1/L2/L3 等级。
- 趋势洞察：分析竞品、趋势、市场声音、拐点和机会判断。
- 成稿：生成公关稿、日报、周报、简报等末端内容。

## 交叉意图处理

- 客户洞察里夹带风险或舆情：以客户洞察为主，只吸收风险 / 舆情的必要字段。
- 风险洞察里需要舆情：舆情只作为外部环境和声誉风险证据，不替代司法、合规、财务等结构化信号。
- 舆情洞察里需要企业主体：先做主体识别，再查舆情，避免品牌名和法人主体混淆。
- 用户只要名单：进入客户洞察的名单筛选，不写成长篇分析。
- 用户只要字段：进入对应方法论的精简字段模式，不生成完整报告。
