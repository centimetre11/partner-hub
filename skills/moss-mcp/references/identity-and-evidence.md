# 主体识别与证据纪律

## 主体识别

- 企业类分析优先取得 18 位统一社会信用代码。
- 用户只给品牌、简称、集团、分公司时，先确认法人主体边界。
- 候选企业多于 1 家时，不要静默选择；列出关键差异让用户确认。
- 境外主体、SPV、分支机构、集团公司要在输出里声明分析口径。

## 易混主体处理

| 情况 | 处理 |
| --- | --- |
| 同名 / 近似主体 | 用行业、地区、股东、经营状态和用户上下文消歧；仍不确定就询问 |
| 分公司 / 支公司 | 明确是查分支自身，还是转到总公司合并判断 |
| 集团 / 控股 | 先说明本次只覆盖哪个法人，不能把集团整体风险硬套到单体 |
| 品牌名 | 做品牌到法人映射，并标注置信度 |
| 境外主体 | 不能传作 18 位信用代码；需锁定境内运营主体或声明结构化采集不可用 |
| 功能型 SPV | 按“小尺度 + 集团协同 + 政策目的”解释，不与母公司规模口径混用 |

## 证据纪律

- 数字、时间、主体关系、风险结论、舆情数量必须来自工具结果。
- 没有证据时，选择补查、弱化措辞并标注未验证，或删除。
- 空结果只能说明当前工具、条件、窗口下未返回记录。
- 结论必须保留局限，例如授权工具不足、数据源失败、窗口未查完。

## 反幻觉自检

输出前检查 8 类内容：状态、关系、数字、来源、法规、资质、时间、工具名。

- 有出处：正常写。
- 没出处：补查、弱化并标注未验证，或删除。
- 工具失败：先重试或降级，仍失败则写入局限，不用猜测填补。
- 权限失败：报告为授权问题，不写成无数据。
