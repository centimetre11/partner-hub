# MCP 使用手册

## 基本流程

1. 调用 `tools/list` 获取当前可用工具。
2. 根据任务目标选择最小必要工具，不为“全面”盲目扩展。
3. 调用 `tools/call`，参数只使用 schema 暴露字段。
4. 读取结构化结果、分页 / 续查提示、错误码和空结果提示。

## 舆情窗口

舆情类工具可能只查询一个有限时间窗口。若实时结果提示仍需继续查询：

- 使用返回结果中暴露的下一窗口参数发起下一次调用。
- 续查参数名称、时间格式和边界以实时工具 schema / 返回结果为准。
- 外部 Agent 或外部系统负责跨窗口合并、去重、按时间排序。

## 失败区分

- `INVALID_ARGUMENT`：修正参数后重试。
- `UNAUTHORIZED` / `FORBIDDEN`：当前 Client 未授权或 Token 不可用。
- `DATA_SOURCE_ERROR`：上游数据源失败，保留已获得证据并声明局限。
- 空结果：说明查询成功但当前窗口/条件无匹配，不得写成“没有风险”。
