# 示例：客户洞察 worked example

> 示例里的事实是演示材料。真实执行时必须以当前 MCP 工具返回为准。

## 用户问题

查一下“帆软软件有限公司”，看看适合从哪里切入。

## 路由判断

- 任务类型：客户洞察。
- `primary_scene`：`company_overview`。
- 用户真实目标：不是要企业百科，而是要销售跟进切入点。
- 需要补充维度：基本盘、行业定位、近期动态、招投标 / 招聘 / 知识产权等商机信号、必要风险信号。
- 不自动展开：完整风险报告、完整舆情趋势、全部专利 / 招投标明细。

## 查询计划

1. 调用实时 `tools/list`，确认当前 Client 有哪些企业检索、画像、动态、招投标、招聘、知识产权、风险信号能力。
2. 先锁定主体：企业全称、统一社会信用代码、经营状态；若同名候选多于 1 家，先让用户确认。
3. 查询企业基本盘：行业、成立时间、注册资本、参保人数或规模代理指标、经营范围。
4. 查询商机信号：近期中标 / 招标、招聘扩张、知识产权 / 资质、公开动态。
5. 只做必要风险扫面：经营异常、处罚、涉诉或负面新闻摘要；不铺九维风险报告。
6. 读取结果后再分析，空结果、未授权、数据源失败分开写。

## 假设关键证据

| 证据 | 解释方式 | 不应误判 |
| --- | --- | --- |
| 企业持续经营多年，参保人数稳定 | 具备稳定组织和交付能力，可支持中大型客户服务 | 不等于业务仍高速增长 |
| 近期有政企 / 大型组织相关中标或新闻 | 说明有预算型客户场景，可从管理驾驶舱、数据治理、报表协同切入 | 单条新闻不等于确定商机 |
| 招聘集中在研发、解决方案、交付岗位 | 可能在强化产品和交付能力，适合从复杂项目需求切入 | 招聘数量不是营收规模 |
| 知识产权 / 资质较多 | 有产品壁垒和准入材料，可用于方案可信度 | 专利多不等于采购意愿强 |
| 未检索到严重红线 | 当前窗口未见强阻断信号 | 空结果不等于无风险 |

## 输出示例

### 结论摘要

帆软软件有限公司更适合从“数据分析平台升级、管理驾驶舱、报表治理和复杂交付能力”切入，而不是泛泛介绍数字化。当前可见信号显示其组织和产品能力较稳定，但仍需要结合目标客户预算、现有系统和采购阶段复核。

### 主体与口径

- 本次以工具识别到的“帆软软件有限公司”法人主体为分析对象。
- 若上游任务实际指集团、分公司、产品品牌或海外主体，需要重新确认口径。

### 关键发现

1. **基本盘稳定**
   成立时间、经营状态、人员规模等基础信号支持“成熟软件厂商”判断。这个结论适合用于客户信任建立，但不能直接推断近期增长。

2. **切入点应围绕管理分析和数据治理**
   若工具返回的业务范围、公开动态、招投标或招聘信号集中在数据分析、报表、管理驾驶舱、解决方案交付，应把话术聚焦到“已有系统整合、指标口径统一、管理层看数效率”。

3. **商机判断要看触发事件**
   中标、招聘扩张、新产品发布、行业活动、客户案例都是触达时机。只有出现明确项目、组织扩张或客户场景变化时，才适合推进深度拜访。

4. **风险信号只作为跟进约束**
   若仅有少量历史诉讼或普通经营异常，不应把客户洞察改成风险报告；只需提示合作前复核合同履约、账期和主体口径。

### 下一步动作

- 若目标是销售跟进：补查近期招投标、招聘、新闻和客户案例，形成 3 个可沟通场景。
- 若目标是找相似客户：转入 `company_similar`，先选择行业、规模、区域或资质作为对标维度。
- 若目标是批量拓客：转入 `company_filter`，把行业、地区、规模和科技标签转成筛选条件。

### 局限

- 当前结论只基于本次 MCP Client 已授权工具。
- 未授权或空结果的维度不能写成事实不存在。
- 若用户需要供应商准入、授信或投资判断，应切换到风险洞察方法论。
