# 舆情洞察方法论

## 核心方法

舆情洞察不是“搜一下网上怎么说”，而是从噪声中识别信号，按“事实 - 推断 - 行动”输出判断。

每次先拆问题：

1. 主体：品牌、企业、产品、人物、事件、竞品或行业。
2. 方向：负面 / 危机、正面 / 口碑、中性 / 趋势。
3. 时效：实时、短窗口、长周期回顾。
4. 意图：要信息、判断、监控、趋势洞察，还是成稿。
5. 产出：口头回复、数据报告、公关稿、日报 / 周报 / 简报。

模糊时问一句；简单的“查一下最近有没有负面”可直接进入检索，但输出必须说明查询窗口和来源范围。

## 四步分析链

```text
              舆情检索
              /      \
    危机识别          趋势洞察
              \      /
              内容成稿
```

- 舆情检索：把自然语言需求翻译成检索策略，负责召回、窗口控制、样本切片和精读入口。
- 危机识别：识别负面、危机等级、攻击论点、传播源、同事件聚合和建议动作。
- 趋势洞察：做竞品并行、时序趋势、市场声音聚类和业务机会判断。
- 内容成稿：只在需要公关回应稿、日报、周报、简报、高管简报等成稿时进入。

这是外部 Agent 的分析步骤，不代表外部存在同名角色或工具。

## 路由规则

| 用户意图 | 路径 | 输出重点 |
| --- | --- | --- |
| 查 / 看某品牌舆情 | 舆情检索；必要时补危机识别 | 查询结果、样本、窗口和局限 |
| 有没有负面、黑稿、投诉、危机 | 舆情检索 → 危机识别 | 等级、攻击论点、传播源、动作 |
| 竞品近期动向、趋势、对比、市场声音 | 舆情检索 → 趋势洞察 | 差异、拐点、机会 |
| 公关稿、回应稿、声明、日报、周报 | 舆情检索 + 危机识别 / 趋势洞察 → 内容成稿 | 可发布初稿或简报 |
| 只整理用户给的现成内容 | 内容成稿或文件生成能力 | 不触发额外舆情查询 |

## 舆情检索

### Step 1：解析需求

- 查询主体：品牌、企业、产品、人物、事件或话题。
- 时间范围：用户指定日期优先；未指定默认近期窗口。
- 任务倾向：负面、竞品、趋势、市场声音或临时事实核验。
- 来源范围：新闻、微信、微博、论坛、短视频、APP 等，以实时工具支持为准。
- 排除词：招聘、股票、无关同名主体、同名品牌等噪声。
- 地域：用户明确提及省市时加入，不要无依据添加。

### Step 2：制定检索策略

- 具体事件、短窗口报告、危机传闻、投诉维权、监管调查：先轻量查证，控制等待和上下文。
- 长周期趋势、品牌全量监控、市场声音洞察、竞品高互动样本：再扩大窗口、分页或分段。
- 不要把多家竞品拼成一个关键词；每家单独查询后对比。
- 简单查询优先用结构化字段，例如 must / should / not；复杂嵌套逻辑才用完整布尔表达式。
- 空格不是 AND；`品牌 负面` 应拆成必须包含品牌和负面词。
- `AND`、`OR`、`AND NOT` 必须大写，前后保留半角空格。
- 关键词太长时拆成多次查询，不硬塞超长表达式。

### Step 3：时间窗口和续查

- 单次工具可能只覆盖有限窗口；超过窗口时按实时返回的续查字段继续。
- 外部 Agent 或外部系统负责跨窗口合并、去重、按时间排序。
- 趋势分析建议按周、月或工具窗口分段；每段记录条数、情感分布、TOP 来源。
- 时间跨度很长时，先抽样判断是否值得扩大，不默认全量拉取。

### Step 4：样本切片

- 危机 / 实时：偏最新采集时间和高互动来源。
- 趋势 / 回溯：偏发布时间或内容时间。
- 传播影响：看点赞、转发、评论、粉丝量和信源等级。
- 拿不准或任务关键时，用两种排序各拉一次再合并去重，减少单一排序盲区。

### Step 5：质量判断

查询后先判断质量再下结论：

- 命中量是否足够。
- 是否被同名主体污染。
- 负面 / 正面 / 中性比例是否可信。
- 是否有权威信源或高互动样本。
- 是否需要精读原文。
- 是否还有下一窗口。

空结果处理顺序：扩大时间窗 → 删除过强排除词 → 扩大来源 → 仍为空时说明“当前条件下未检索到”，不能写“没有舆情风险”。

## 精读纪律

摘要不是全文。以下场景必须精读再下结论：

- L1 / L2 危机判定。
- 公关回应稿和攻击论点提取。
- 转发链 / 二次传播判断。
- 长文新闻或深度调研稿。
- 高互动、权威信源或严重指控内容。

精读时先锁定 1-3 条关键条目；按实时 schema 使用可用的精读、条目读取或结果内检索能力。不要直接读取整份大文件，也不要把摘要当原文。

## 危机识别

### 危机评分思想

危机不是数量越多越高，而是看紧迫度。综合三类权重：

1. 内容权重：权威媒体、监管信源、头部财经媒体、认证账号权重更高。
2. 传播权重：粉丝量、点赞、评论、转发、跨平台扩散。
3. 情感烈度：高敏感负面、明确指控、投诉维权、监管调查、诈骗跑路等强词。

同一事件多条内容要聚合，不要把转发和转载当作独立危机。

### L 级判定

| 等级 | 判断 | 后续动作 |
| --- | --- | --- |
| L1 | 业务正在被伤害，分钟级响应；高敏负面 × 权威信源 × 仍在传播 | 立即提取攻击论点、传播源和回应建议 |
| L2 | 信号在扩散，24 小时内响应；中等负面在多平台聚集 | 纳入待跟进，补精读和传播观察 |
| L3 | 零星不利信号，未形成传播 | 记录并监控，不夸大 |

10 条 L3 不等于 1 条 L1；看性质、信源、传播速度和互动放大。

### 同事件聚合

按三个特征综合判断：

- 标题 / 正文相似度高。
- 核心实体一致：主体名 + 关键事件词一致。
- 时间临近：发布时间集中。

聚合后输出：

- 代表条目。
- 成员数量。
- 时间跨度。
- 最高危机等级。
- TOP 信源。
- 攻击核心论点。
- 增速或扩散迹象。

### 攻击论点提取

只对 L1 / L2 或用户要求回应的内容做。

输出攻击论点时区分：

- 事实指控：对方声称发生了什么。
- 情绪标签：如“割韭菜”“欺骗消费者”等。
- 证据状态：是否有截图、官方文件、媒体报道、用户原话。
- 可回应点：能用数据、案例、流程、时间线回应什么。
- 不宜回应点：缺证据、法律敏感、容易扩大传播的内容。

## 趋势洞察

### 模式 A：竞品并行

- 每个竞品单独查询，不拼成一个关键词。
- 横向比较声量、情感、TOP 内容、关键事件和差异点。
- 竞品数量较多时，提示耗时、窗口限制和样本口径。
- 价值在差异和机会，不在罗列每家新闻。
- 输出威胁等级时必须给证据，不主观写“竞品不行”。

### 模式 B：时序趋势

- 按周、月或工具窗口分段查询。
- 每段记录总量、情感分布、TOP 来源、关键主题。
- 识别拐点：声量突增、负面占比上升、权威信源介入、平台迁移。
- 拐点解释：补查拐点窗口的高互动内容，找驱动因素。
- 数据不足时说“不足以判断趋势”，不要硬画趋势。

### 模式 C：市场声音

- 优先读取高互动用户内容和原话。
- 归纳真实诉求：用户想要什么。
- 归纳痛点：用户当前不满意什么。
- 识别机会：哪些诉求 / 痛点未被主流产品充分满足。
- 可用公开行业资料做对照，但必须区分“舆情数据”和“公开资料”。

### 综合输出

结论分三层：

1. What：发生了什么。
2. So What：对客户意味着什么。
3. Now What：客户可以怎么做。

跨模式综合时，用“一句话本期总结 → 最大发现 → 竞品段 → 趋势段 → 市场声音段 → 综合建议”组织。

## 内容成稿

仅在用户明确要求稿件、日报、周报、简报或高管版本时进入。

成稿前先沉淀事实、攻击论点、证据状态、可回应点和局限。所有稿件都必须标注“初稿，需人工审核后发布”。不能编造数据，缺素材时用待核实占位或删掉。

## 异常与降级

| 情况 | 处理 |
| --- | --- |
| 查询返回 0 条 | 扩大时间窗、放宽排除词、扩大来源；仍为空则说明当前条件无结果 |
| 授权失败 | 说明当前 Client 未授权或凭证不可用，不写成无数据 |
| 限流 / 超时 | 保留已获取样本，标注数据采集不完整，建议稍后重试 |
| 品牌词过于通用 | 询问更精准限定词或补排除词，不继续大规模查询 |
| 行业背景缺失 | 询问行业后再选负面词集合 |
| 仅公网搜索可用 | 标注“数据来源受限，仅供参考”，不能冒充独家舆情统计 |

## 高频误判清单

| 误判 | 正确做法 |
| --- | --- |
| 负面数量多就判危机 | 先看信源、传播力、情感烈度和是否同事件聚合 |
| 10 条 L3 当作 1 条 L1 | L1 看紧迫度和业务伤害，不看简单条数相加 |
| 摘要当全文 | 严重指控、回应稿、危机评级前必须精读 |
| 多家竞品拼一个关键词 | 每家单独查，再横向对比 |
| 空结果写成没有风险 | 只能写当前窗口和来源下未检索到 |
| 公网搜索混进舆情统计 | 公开搜索只能补背景，不能冒充舆情样本口径 |
| 长窗口一次查完 | 按工具窗口分段，合并、去重、按时间排序 |
| 同名品牌污染结果 | 加行业、地区、产品、排除词，必要时先澄清主体 |

## 输出字段

快速判断可输出：

```text
opinion_level: L1 / L2 / L3 / 未知
sentiment: 正面 / 中性 / 负面 / 混合 / 未知
key_topics: 话题 1；话题 2；话题 3
spread_sources: 平台或信源摘要
query_window: 查询时间窗
coverage: 来源范围和样本量
confidence: 高 / 中 / 低
recommended_action: 动作 1；动作 2
```

字段无法判断时写未知并说明缺什么数据；公网搜索只能作为补充，不能冒充独家舆情判断。

## 输出纪律

- 事实、推断、行动分层；事实可追溯，推断说明依据，行动要可执行。
- 差异先行：竞品 / 趋势的价值是差异、拐点和驱动因素，不是新闻罗列。
- 机会落地：每个发现回答“对客户意味着什么，可以做什么，什么时候重新评估”。
- 客观描述竞品，不主观定性“竞品不行”。
- 数据不足时降级表述，例如“样本不足，只能初步观察”；不要硬给趋势或等级。
- 负面舆情未交叉验证时，必须标注“公开舆情信号，未能验证真实性”。
