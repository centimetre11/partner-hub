# 风险洞察方法论

## 核心方法

风险洞察不是“查到了几条负面”，而是把工商、司法、财务、合规、关联方、外部环境等事实信号转成可决策、可溯源、边界清晰的风险判断。

每个风险结论必须回答五件事：

1. 评估谁：法律主体、集团 / 分支 / 品牌 / 境外主体边界。
2. 站在哪个业务场景：供应商准入、客户信用、投资尽调、持续监控或通用分析。
3. 哪些事实触发：证据、时间、来源、当前状态。
4. 风险如何传导：影响履约、账期、准入、投资、声誉还是合规。
5. 下一步怎么处理：复核动作、控制条件、监控触发器。

## 九维框架

| 维度 | 看什么 | 关键判断 |
| --- | --- | --- |
| 经营稳定性 | 存续年限、工商变更、招投标、招聘、年报 | 是否持续经营、是否突然收缩、业务是否断崖 |
| 治理与人事 | 法人、董监高、股东结构、关键岗位变动 | 治理是否稳定，是否存在逃废债或控制权异常 |
| 财务健康 | 上市财务、年报、税务、现金流代理指标 | 偿付能力、盈利质量、现金压力 |
| 合规处罚 | 行政处罚、经营异常、严重违法、资质证书 | 是否触发准入或合规红线 |
| 司法风险 | 诉讼、执行、失信、限高、终本、案由分布 | 是历史噪声还是当前履约能力受损 |
| 关联方风险 | 股东、对外投资、实控人版图、担保链 | 风险是否从关联方传导到目标主体 |
| 资产股权 | 股权出质、冻结、动产抵押、知识产权、融资 | 资产是否被质押、冻结、处置或异常融资 |
| 外部环境 | 监管、行业政策、制裁、重大舆情、行业暴雷 | 结构化数据看不到的系统性风险 |
| 特殊信号 | 清算、注销、吊销、功能型主体、政策兜底 | 是否需要切换评估尺度 |

不要把九维当固定目录硬填。浅度任务可只覆盖关键维度；深度 / 决策类任务才尽量覆盖全维。

## Phase 1：目标对象锁定

一次性锁定主体、边界、视角和场景。

### 主体识别

- 优先使用企业全称 + 18 位统一社会信用代码。
- 同名主体多于 1 家时，不静默选择；列出地区、行业、状态、股东、规模差异让用户确认。
- 品牌名、简称、产品名要映射到法人主体，并标注置信度。
- 境外主体不能硬套境内统一社会信用代码；需锁定境内运营主体或声明数据能力限制。

### 七类边界探针

| 类型 | 识别信号 | 处理方式 |
| --- | --- | --- |
| 同名同根 | 多家名称相近、行业或地区接近 | 消歧后再分析 |
| 总分公司 | 分公司、支公司、营业部 | 明确分析分支自身还是转总公司 |
| 集团群 | 集团、控股、关联企业众多 | 先声明本次覆盖哪些法人 |
| 品牌名不等于法人 | 用户给品牌 / 产品 / 简称 | 做品牌到法人映射 |
| 壳 / 通道 / 历史主体 | 注册多年但公开信息稀薄或业务迁移 | 查实际业务承载主体 |
| 跨境同名 / 同业 | Holdings、Ltd、境外注册地 | 做跨管辖区边界声明 |
| 功能型主体 | SPV、持牌通道、税收洼地、集团隔离主体 | 切“小尺度 + 政策目的 + 集团协同”口径 |

输出必须先说明：“本次以谁为评估对象，包含 / 不包含哪些主体。”

## Phase 2：意图识别

识别两个正交参数。

### 分析模式

| 模式 | 触发判断 | 输出深度 |
| --- | --- | --- |
| 浅度 | “看一下 / 查一下 / 了解一下”，无明确决策场景 | 快速风险画像，仍要有事实触发原因 |
| 深度 | 尽调、准入、授信、投资、全面评估、支撑决策 | 九维扫描 + 必要增强分析 |
| 监控 | 监控、预警、订阅、持续跟踪 | 监控对象、触发器、复核动作 |

### 聚焦类型

| 聚焦 | 判断 | 处理 |
| --- | --- | --- |
| 全维 | 用户未强调聚焦或要求全面 | 覆盖九维，按风险强度排序 |
| 指定维度 | 只问财务、司法、处罚等少数维度 | 只分析指定维度，必要时补强支撑维度 |
| 全维 + 增强 | 全面分析且特别关注某维度 | 全维同深度做，增强维度展开因果链 |

正式采集前应给出简短路径预告：评估视角、模式、重点维度、可能的数据缺口。

## Phase 3：数据采集

按模式缩放，不盲目“全查”。

### 采集范围

- 浅度：基础信息 + 异常 / 处罚 / 司法 / 人员 / 关联等 4-6 个关键维度。
- 深度：尽量覆盖九维，包含司法明细、关联方、外部环境。
- 监控：基础 + 异常 + 最新新闻 / 处罚 / 司法摘要 + 触发器。

所有工具调用必须来自实时 `tools/list`。参数、分页、续查、返回字段都以实时 schema 为准。

### 采集后必读

不能只调用采集能力就直接下结论。采集后必须读取：

- 事实摘要：每个维度的核心结论和证据。
- 覆盖状态：成功、空结果、不适用、失败分别处理。
- 时间锚点：状态当前是否仍在列，事件发生在什么时候。
- 明细入口：高价值维度是否需要读取明细或样本。

### 外部环境校读

结构化数据之外，要补结构化能力看不到的信号：

- 制裁、出口管制、监管处罚、行业整顿。
- 重大事故、暴雷、退市、重组、供应链断裂。
- 知名企业和时效性强场景必须核验近期重大事件。
- 外部资料要和结构化数据分层，不可混成同一来源。

### 制度型画像

Phase 3 末尾锁定画像，Phase 4 按画像校准。

| 画像 | 识别信号 | 校准要点 |
| --- | --- | --- |
| 城投 / 融资平台 | 国资股东 + 城建 / 城投 / 建投等属地化经营 | 用化债政策、区域财政、再融资通道校准 |
| 上市公司 | 有证券代码或上市主体关联 | 看扣非、质押平仓线、公告披露，不只看净利润 |
| 民营非上市 | 自然人控股、非国资、非上市 | 用参保、招投标、纳税、司法做代理 |
| 国企非城投 | 国资委 / 央企集团参股 | 多层嵌套和人事调动可能是制度，不直接告警 |
| 外资 / 合资 | 境外股东、VIE、跨国集团 | 关联交易和转移定价需结合母公司全球风险 |
| 政策兜底主体 | 政策性银行、电网、铁路、烟草、政策性担保 | 用政策延续性替代市场化偿债能力 |
| 功能型主体 | SPV、通道、税洼、业务隔离子公司 | 小规模和信息少未必是壳，要看持牌 / 政策目的 |

## Phase 4：分析校准

### 三向校准

每个参与维度都按三向校准：

1. 事实信号：发生了什么，是否仍在，证据强度。
2. 制度型画像：这个信号在该主体类型下是常态还是异常。
3. 外部环境：行业、政策、近期事件是否放大或缓释。

数据反常但符合主体画像时，不直接告警；外部已有重大信号而结构化数据未显示时，必须补入局限和判断。

### 反直觉陷阱

| 看到 | 错误直觉 | 正确处理 |
| --- | --- | --- |
| 经营异常 | 经营一定出问题 | 多数是未按时年报等工商合规问题，看原因和是否在列 |
| 涉诉总数高 | 全部都是风险 | 分原告 / 被告 / 被执、已结 / 未结、金额和案由 |
| 被执行总额 | 当前欠这么多钱 | 区分历史累计和当前未履行，看终本与失信 |
| 注册资本变更 | 变了就是风险 | 增资、减资、认缴转实缴含义不同，减资更需关注 |
| 法人变更 | 肯定有问题 | 看时间窗和是否与地址、股东、经营范围连环变化 |
| 股权出质 | 一定缺钱 | 看质权人：银行融资、关联方质押、自然人质押含义不同 |
| 限制高消费 | 已经破产 | 限高是司法措施；限高 + 失信 + 终本叠加才强红线 |
| 没有数据 | 没有风险 | 大规模或存续多年企业零公开信息反而可疑 |
| 国资 / 城投负债高 | 一律高风险 | 结合政策兜底、区域财政、再融资通道校准 |
| 上市公司净利润高 | 风险低 | 看扣非、现金流、质押、关联交易和公告风险 |
| 关联方有风险 | 目标主体必然高风险 | 只有控股、担保、资金往来或业务依赖明确时才强传导 |
| 小规模主体信息少 | 一定是壳 | 先判断是否为功能型主体、持牌通道或税收 / 业务隔离 SPV |

### 红线筛查

强红线包括但不限于：

- 注销、吊销、清算、破产。
- 失信被执行、限制高消费、终本案件、重大未履行执行。
- 严重违法失信、重大行政处罚、税务非正常。
- 重大安全 / 质量 / 数据合规事故。
- 涉及业务准入的一票否决资质失效。

红线不阻断分析，仍需说明原因、传导、影响和缓释项。

### 增强分析

深度模式或用户明确聚焦增强时启动：

1. 跨维矩阵：识别多类信号共振，例如司法 × 关联方、治理 × 法律、经营 × 外部环境。
2. 风险传导：从股东、实控人、担保链、上下游、集团资金链解释风险如何传到目标主体。
3. 暴雷模式：匹配流动性、财务造假、实控人、行业崩、扩张、担保链、供应链等特征链。
4. 风险演进：按 0-3 / 3-6 / 6-12 个月给修复、恶化、突变三类路径。

## Phase 5：输出

综合判断按四条规则合成：

1. 红线优先。
2. 多维共振上抬。
3. 场景重点维度设下限。
4. 同级无证据不上抬。

### 风险等级

对外可使用：极高 / 高 / 中高 / 中低 / 低 / 待核实。若上游只接受三档，可映射为高 / 中 / 低 / 未知。

等级必须由事实触发原因支撑。数据缺口要单列，不要用“未查到”替代“无风险”。

### 固定字段

```text
risk_level: 极高 / 高 / 中高 / 中低 / 低 / 待核实
trigger_reasons: 原因 1；原因 2；原因 3
risk_path: 风险如何影响准入、账期、合作或投资
confidence: 高 / 中 / 低
recommended_action: 复核动作和业务动作
```

### 场景化分寸

| 场景 | 必含 |
| --- | --- |
| 供应商准入 | 履约能力、资质有效性、应核实清单、准入条件 |
| 客户信用 / 授信 | 偿付能力、账期建议、应收回收风险、监控触发器 |
| 投资尽调 | 关键风险点、退出路径、估值假设风险、补充尽调事项 |
| 持续监控 | 监控对象、指标、触发阈值、复核频率 |
| 通用排查 | 风险事实、因果链、合作前应核实信息 |

## 边界

- 不提供个人征信判断。
- 不替用户决定“合作 / 不合作”，只给风险和控制条件。
- 不传播未经核验的负面舆情。
- 法务条款、估值建模、行业研究专题可以提供风险事实，但不替代专业结论。
- 输出前自检：数字、状态、关系、来源、法规、资质、时间都必须有出处；没有出处就补查、弱化或删除。
